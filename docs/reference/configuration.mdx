# Configuration Reference

All configurable environment variables across Sukko services. Values are set via `env:` struct tags in Go config files ‚Äî the `envDefault` is the source of truth for defaults.

## Upgrade Notes

### Per-Tenant Broadcast Channels (v0.x ‚Üí current)

The `VALKEY_CHANNEL` configuration is now a **channel name prefix**. Each tenant's messages are published to `{VALKEY_CHANNEL}:{tenantID}` instead of a single shared channel. This change is not backward-compatible with older pod versions.

**Rolling deploy compatibility**: This change is **not safe to roll out as a rolling deploy** when upgrading from a legacy deployment that used a single shared `ws.broadcast` channel. Old pods publish to `ws.broadcast`; new pods subscribe to `ws.broadcast:acme`. During a rolling deploy, messages published by old pods will not reach new pods and vice versa. Use a **coordinated deployment** (stop all pods, then start the new version) when upgrading from a legacy single-channel deployment. New deployments (no existing pods) are unaffected.
