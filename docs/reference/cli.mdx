# CLI Reference

The `sukko` CLI manages your Sukko deployment — tenants, keys, testing, and local development.

## Install

```bash
# macOS / Linux
brew install klurvio/tap/sukko

# Windows
scoop bucket add sukko https://github.com/klurvio/scoop-bucket
scoop install sukko
```

### `sukko completion [bash|zsh|fish]`

Generate shell completion scripts

Generate shell completion scripts for sukko.

To load completions:

Bash:
  $ source &lt;(sukko completion bash)
  # To load for each session, add to ~/.bashrc:
  $ echo 'source &lt;(sukko completion bash)' &gt;&gt; ~/.bashrc

Zsh:
  $ source &lt;(sukko completion zsh)
  # To load for each session, add to ~/.zshrc:
  $ echo 'source &lt;(sukko completion zsh)' &gt;&gt; ~/.zshrc

Fish:
  $ sukko completion fish | source
  # To load for each session:
  $ sukko completion fish &gt; ~/.config/fish/completions/sukko.fish

### `sukko test`

Test WebSocket connectivity to the gateway (CLI-only, no tester needed)

Test WebSocket connectivity by opening a single connection directly from the CLI.

Verifies: WebSocket upgrade (101), subscribe, and basic message flow.
For full deployment validation (Kafka pipeline, multi-tenant isolation),
use 'sukko test smoke' which orchestrates the tester service.

**Flags:**

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--api-key` | string | — | API key for authentication |
| `--gateway-url` | string | — | Gateway URL (overrides context) |
| `--timeout` | duration | `10s` | Connection timeout |
| `--token` | string | — | JWT token for authentication |

### `sukko create [name]`

Create a context interactively

Create a named context for a Sukko environment. Prompts for required
fields (name, provisioning URL, gateway URL, admin token) and optional fields.
Secrets are entered with masked input to avoid shell history exposure.

All fields can also be provided as flags for non-interactive/CI use.

**Flags:**

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--admin-token` | string | — | Admin authentication token |
| `--api-key` | string | — | API key for gateway access |
| `--environment` | string | — | Environment name (e.g., staging, production) |
| `--force` | bool | `false` | Overwrite existing context without confirmation |
| `--gateway-url` | string | — | WebSocket gateway URL (e.g., wss://ws.example.com) |
| `--provisioning-url` | string | — | Provisioning API URL (e.g., https://api.example.com) |
| `--tenant` | string | — | Default active tenant ID |
| `--tester-url` | string | — | Tester service URL |

### `sukko edition`

Show current edition, limits, and usage

Show the current Sukko edition (Community/Pro/Enterprise), license status,
hard limits, and live resource usage.

Fetches data from provisioning's /edition endpoint (tenant usage) and
gateway's /edition endpoint (connection/shard usage), merging both.
Falls back to locally stored license key if the platform is unreachable.

---

### `sukko compare`

Compare Community, Pro, and Enterprise editions

Display a comparison table of all three Sukko editions.

This command works offline — the comparison data is hardcoded from the
published edition matrix. No running services required.

### `sukko up`

Start the local development environment

Start the local development environment using Docker Compose.

Reads .sukko/config.json (created by 'sukko init'), activates the selected
infrastructure profiles, waits for all services to become healthy, and
provisions a default tenant for immediate use.

**Flags:**

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--pull` | bool | `false` | Always pull latest images before starting |
